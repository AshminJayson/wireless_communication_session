03Part One

Evolution of Mobile Networks

On screen
  • Opener. Thesis: “Every generation existed to solve the hard limit of the one before it.”
Learn the tech

The honest engine behind every generation is spectral efficiency — bits per second squeezed out of each hertz of spectrum (bits/s/Hz). You can grow capacity three ways: more spectrum (wider bands), better efficiency (smarter modulation/coding), or more spatial reuse (smaller cells, more antennas). Each generation pulled all three levers harder than the last.

Plant the thread that carries the whole talk and call back to it on every later slide.

05Evolution · The signal · ANCHOR

One signal, five generations

On screen
  • 1G 1980s analog voice · 2G 1990s digital + SMS · 3G 2000s mobile internet · 4G 2010s LTE broadband · 5G 2020s connected ecosystem.
Learn the tech — what actually changed each generation

1G — Analog, FDMA (each call gets its own frequency channel). Voice only, circuit-switched (a dedicated line held for the whole call). Systems: AMPS, NMT.

2G — First digital generation. TDMA (GSM) and CDMA (IS-95) let many users share spectrum by time slots or codes. Added SMS; digitized voice = clearer, more secure, more capacity. Packet data bolted on later as GPRS/EDGE (“2.5G”).

3G — WCDMA / UMTS. Packet-switched data becomes first-class → real mobile internet. HSPA pushed it to tens of Mbps.

4G — LTE, built on OFDMA (data split across thousands of narrow orthogonal subcarriers — robust against interference, very efficient). Fully all-IP: even voice is data (VoLTE). MIMO introduced. ~100 Mbps–1 Gbps.

5G — NR (New Radio). Keeps OFDM but adds flexible numerology (selectable subcarrier spacing) and two frequency ranges: FR1 sub-6 GHz and FR2 mmWave (24 GHz+). Massive MIMO + beamforming. Built for three very different traffic types at once (see slide 9).

The one-line story: the access method evolved FDMA → TDMA/CDMA → WCDMA → OFDMA, letting each generation pack more independent users and bits into the same spectrum.

Talk track

The deck’s anchor — slow down. Voice → text → internet → video → machines. Point at the denser waveforms: each carries a new kind of information, not just more speed.

If short on time: protect this slide.
06Evolution · Drivers

Why networks never stand still

On screen
  • Growing data consumption · explosion of devices (IoT) · real-time expectations (gaming, AR/VR, control) · automation & M2M.
Learn the tech

These four map onto measurable network demands: throughput (data volume), connection density (devices/km²), latency (real-time response), and machine-type traffic (lots of tiny, bursty messages). A single network optimized only for fast phones can’t serve all four — which is exactly the problem 5G’s three service classes solve.

Talk track

These forces compound, so demand never settles. Connected devices already outnumber people.

07Evolution · Perspective

What the leaps actually feel like

On screen — download a 2-hour movie (~5 GB)
NetworkTimeThroughput
3G2–3 hours~2–5 Mbps

5GSeconds to a minutehundreds of Mbps to Gbps
Learn the tech — the maths so you’re not caught out

Speeds are in megabits (Mb), file sizes in megabytes (MB). 1 byte = 8 bits, so divide by 8. A 5 GB file = 40,000 megabits. At 50 Mbps: 40000 ÷ 50 = 800 s ≈ 13 min. At 1 Gbps: ≈ 40 s. Real downloads are slower than peak (overhead, shared cells, server limits) — which is why the slide gives ranges, not single numbers.

Talk track

A benchmark people feel. Show of hands — who remembers waiting on a 3G download? Bridge: expectations now assume instant, and that pressure drives the next generation.

095G · Definition

What is 5G?

On screen
  • Fifth generation — an intelligent connected ecosystem. Objectives: extreme speed/capacity, ultra-low latency, massive connectivity, mission-critical reliability.
20 Gbps peak (ITU IMT-2020) ~1 ms air-interface latency 10⁶ / km² device density
Learn the tech — the three service classes (the heart of 5G)

eMBB — enhanced Mobile Broadband. Raw speed and capacity: 4K/8K video, AR/VR, crowds in stadiums. Optimizes for throughput.

URLLC — Ultra-Reliable Low-Latency Communications. ~1 ms latency and 99.999% reliability: remote surgery, factory robots, vehicle control. Optimizes for latency + reliability.

mMTC — massive Machine-Type Communications. Up to ~1 million low-power devices per km²: sensors, meters, trackers. Optimizes for density + battery life.

These three pull in opposite directions, so 5G is designed to run all three over one shared infrastructure at once — that’s the real innovation. The three big numbers on the slide are the ITU IMT-2020 targets (the official requirements 5G had to meet), not everyday handset figures.

Talk track

Stress that 5G is a platform, not a faster phone. Name the three classes plainly. Flag the numbers as ideal targets — saying so builds credibility.

Learn the tech — why latency drops and density rises

Lower latency comes from: shorter transmission slots via flexible numerology (wider subcarrier spacing = shorter symbols), mini-slots (send without waiting for a slot boundary), and edge computing / MEC (put servers near the user so data travels less distance). Higher density comes from massive MIMO serving many users spatially, plus features designed for tiny IoT messages. Note peak speed is the least important row for industry — latency and reliability enable control loops.

Talk track

Walk the rows. The game-changers for industry are latency and density, not peak speed. Note figures are targets/typical and vary by deployment.

115G · Air interface

Massive MIMO & beamforming

On screen
  • Massive MIMO — many antennas per cell, serves many users at once, more capacity/coverage.
  • Beamforming — focused, steerable signals aimed per user; less waste, better signal & efficiency.
Learn the tech — your home turf, go one level deeper

MIMO = Multiple-Input, Multiple-Output: antennas at both ends. Spatial multiplexing sends independent data streams on the same frequency and time, separated only by their spatial path — so N antennas can multiply throughput.

Massive MIMO = a base-station array of 32/64/128+ elements. With MU-MIMO it serves many users simultaneously on the same resources by separating them in space.

Beamforming = applying per-element phase/amplitude weights (precoding) so the wavefronts add up constructively toward one user and cancel elsewhere — a steerable spotlight vs. a floodlight. Flavours: analog (phase shifters, one beam, cheap), digital (per-element control, many beams, power-hungry), hybrid (the practical mmWave choice).

It all depends on channel state information (CSI). In TDD, uplink and downlink share a frequency, so the base station estimates the downlink channel from uplink pilots (reciprocity). The resulting array/beamforming gain raises SNR — essential to overcome mmWave path loss.

Talk track

MIMO = more antennas, parallel streams. Beamforming = aiming, not broadcasting — floodlight vs. spotlight. This is your world; go off-script with a real example.

125G · Architecture & standards

An architecture defined by 3GPP

On screen
  • 3GPP standardizes mobile networks. Chain UE → NG-RAN → 5G Core → Data Network, with reference points Uu, N2/N3, N6. Releases: Rel-15 (first 5G, 2018), NR, 5GC, NSA → SA.
Learn the tech — 3GPP, Releases, and NSA vs SA

3GPP (3rd Generation Partnership Project) is a partnership of regional standards bodies that writes the specs every vendor builds to — so a phone works on any operator. Work ships in numbered Releases: Rel-15 first 5G NR (2018); Rel-16/17 add URLLC, industrial IoT, RedCap, satellite (NTN); Rel-18+ is branded 5G-Advanced.

NSA (Non-Standalone): 5G NR radio bolted onto the existing 4G LTE + 4G core — fast to launch, but no full 5G features. SA (Standalone): 5G NR + the new 5G Core — unlocks slicing, URLLC, the works. Most early networks launched NSA and are migrating to SA.

Reference points are just named interfaces: Uu = the air link UE↔gNB; N2 = gNB↔AMF (control signalling); N3 = gNB↔UPF (user data); N6 = UPF↔internet.

Talk track

5G isn’t one vendor’s design — it’s a shared spec in numbered Releases. The arrow labels are the real 3GPP reference points. NSA reused the 4G core; SA is the full 5G core. Walk left to right.

135G · The three domains

UE, RAN & 5G Core

On screen
  • UE — device + 5G modem + (U)SIM. NG-RAN — gNB radio, Massive MIMO, split CU/DU/RU. 5GC — cloud-native, service-based; AMF/SMF/UPF; control/user-plane split.
Learn the tech — what each domain really contains

UE (User Equipment) — the device, its 5G modem, and the (U)SIM that holds its identity. Cavli’s cellular IoT modules are UE. It speaks NR over the Uu air interface.

NG-RAN — the radio. The base station (gNB) is functionally split: RU (Radio Unit: RF + antennas), DU (Distributed Unit: real-time lower layers), CU (Centralized Unit: higher layers, can be centralized/virtualized). The links are fronthaul (RU–DU), midhaul (DU–CU), backhaul (CU–core). O-RAN opens these interfaces so you can mix vendors.

5G Core (5GC) — cloud-native Service-Based Architecture: software network functions talk via web-style APIs. Key ones: AMF (access & mobility), SMF (sessions), UPF (forwards user data), plus UDM (subscriber data), AUSF (authentication), PCF (policy), NRF (function discovery).

CUPS (control/user-plane separation) lets the UPF sit at the edge near users. Together with SBA, this is what makes network slicing possible — multiple logical networks (one tuned for eMBB, one for URLLC, etc.) on one shared physical network.

Talk track

UE is your world. RAN is the radio (gNB, now CU/DU/RU). The Core is cloud-native — AMF/SMF/UPF. The control/user-plane split is what enables slicing and edge breakout.

155G · The hard parts

What still has to be solved

On screen
  • Challenges: infrastructure cost · spectrum management · security · rural coverage. In production: Private 5G in manufacturing; 5G healthcare.
Learn the tech — spectrum bands and the cost/coverage trade-off

5G spans three spectrum tiers, and they trade capacity against reach: low-band (<1 GHz) travels far and through walls but carries little; mid-band (1–6 GHz, e.g. 3.5 GHz) is the workhorse balance; mmWave (24 GHz+) carries enormous data but only over short range and is blocked by walls, rain, even foliage. mmWave therefore needs dense small cells — which is the main cost driver.

Security: more devices = a bigger attack surface, but 5G also improves it — your permanent identity (SUPI) is sent encrypted as a SUCI, defeating the IMSI-catcher attacks that plagued older networks, with stronger mutual authentication (5G-AKA).

Talk track

Be honest about cost, spectrum, security, rural. Then balance it: Private 5G is a real, growing segment — a concrete example of where investment flows.

176G · The next leap

Why 6G, and what to expect

On screen
  • Why: today’s networks cap out; relentless data growth; immersive AI-driven experiences 5G can’t fully serve.
  • Expected: ~1 Tbps (research target), near-zero/sub-ms latency, AI-native, enhanced end-to-end reliability.
Learn the tech — IMT-2030 and “AI-native”

Just as 5G = IMT-2020, 6G is being defined as IMT-2030 by the ITU (vision framework published 2023). AI-native is the key phrase: in 5G, AI is bolted on for optimization; in 6G the intent is to design the air interface and network around AI from the start — ML doing channel estimation, resource scheduling, and self-management. Treat ~1 Tbps as an aspirational research figure, not a committed spec.

Talk track

Headline: 6G isn’t just faster — it’s AI-native with near-zero latency. Frame ~1 Tbps as aspirational to stay credible.

186G · Enabling technologies

What makes 6G possible

On screen
  • AI in the network · Terahertz comms · Space–air–ground networks · Digital twins.
Learn the tech — the four enablers, briefly but correctly

Terahertz (0.1–10 THz) — enormous bandwidth, so huge data rates, but brutal path loss and molecular/atmospheric absorption mean very short range; needs ultra-dense cells and advanced antenna arrays (a hardware frontier).

AI-native networking — ML embedded in radio resource management and the air interface itself (see slide 17).

Space–air–ground / NTN — Non-Terrestrial Networks: LEO satellite constellations, high-altitude platforms (HAPS), and drones integrated with ground networks for true global coverage (5G already started this in Rel-17).

Digital twins — live virtual replicas of a factory or city, fed by real-time network data, to simulate and optimize the physical system. Worth knowing two more 6G buzzwords: RIS (programmable surfaces that steer reflected signals) and ISAC (the network senses its environment, radar-like, while communicating).

Talk track

Four enablers. THz and space–air–ground are the most physically novel and the hardest hardware problems — your wheelhouse. Digital twins bridge to industry.

206G · Where we are

Research & standardization

On screen

Learn the tech — the standardization timeline

Rough roadmap: ITU-R IMT-2030 vision done 2023 → detailed technical requirements ~2026–27 → 3GPP 6G study items in Rel-20 (~2025–26) → first 6G specs around Rel-21 (~2028) → commercial launches ~2030. Glossary & acronym cheat-sheet

One-page revision. The starred (★) terms are the ones to know cold.

Generations & access methods
FDMA
Frequency-Division Multiple Access — each user gets a separate frequency. (1G)
TDMA
Time-Division Multiple Access — users share a frequency in time slots. (2G/GSM)
CDMA
Code-Division Multiple Access — users share spectrum via unique codes. (2G/3G)
OFDMA ★
Orthogonal Frequency-Division Multiple Access — data spread over many narrow orthogonal subcarriers; the basis of 4G & 5G.
GSM / UMTS / LTE
The dominant 2G / 3G / 4G standards, respectively.
NR ★
New Radio — the 5G air interface (the radio standard itself).
VoLTE
Voice over LTE — carrying voice as data packets on 4G (no separate circuit).
5G service classes & targets
eMBB ★
enhanced Mobile Broadband — high speed/capacity (video, AR/VR).
URLLC ★
Ultra-Reliable Low-Latency Comms — ~1 ms, 99.999% (surgery, robots, vehicles).
mMTC ★
massive Machine-Type Comms — huge density of low-power IoT devices.
IMT-2020
The ITU requirement set that defines “5G” (IMT-2030 = 6G).
ITU / 3GPP ★
ITU = UN body setting the targets; 3GPP = the partnership writing the actual specs (in Releases).
Radio layer
MIMO ★
Multiple-Input Multiple-Output — many antennas sending parallel spatial streams.
Massive MIMO
Base-station array of 32–128+ elements serving many users at once.
MU-MIMO
Multi-User MIMO — separating simultaneous users by their spatial path.
Beamforming ★
Steering signal energy toward a user via per-antenna phase/amplitude weights (a spotlight, not a floodlight).
Precoding
The weights applied across antennas to shape the beam.
CSI
Channel State Information — what the radio knows about the channel; needed to beamform.
TDD / FDD
Time- vs Frequency-Division Duplex — uplink & downlink share time slots vs separate frequencies.
FR1 / FR2
5G frequency ranges: FR1 = sub-6 GHz, FR2 = mmWave (24 GHz+).
mmWave
Millimetre-wave spectrum — huge capacity, short range, easily blocked.
